1. Replace the stub `internal/server/server.go` with full implementation:

   **Server struct:**
   ```go
   type Server struct {
       emberclaw.UnimplementedPicoClawServiceServer
       agent     AgentProcessor
       startTime time.Time
       model     string    // from config, for Status RPC
       provider  string    // from config, for Status RPC
       ready     bool
   }
   func New(agent AgentProcessor) *Server { ... }
   ```
   Add setter methods: `SetModel(string)`, `SetProvider(string)`, `SetReady(bool)`.

   **Chat handler (GRPC-02, GRPC-05):**
   - Follow RESEARCH.md Pattern 2 exactly
   - Assign `sessionKey = "grpc:" + uuid.New().String()` at stream start
   - If first ChatRequest has non-empty SessionKey, use that instead (session resume)
   - Loop: Recv -> ProcessDirect(stream.Context(), msg, sessionKey) -> Send response
   - Handle io.EOF (client done) and context cancellation (Pitfall 4: always use stream.Context())
   - Return gRPC status codes: codes.Internal for agent errors, codes.Canceled for stream close

   **Query handler (GRPC-03):**
   - Generate a one-time session key: `"query:" + uuid.New().String()`
   - If request has SessionKey, use that instead
   - Call `ProcessDirect(ctx, req.Message, sessionKey)`
   - Return QueryResponse{Text: response}
   - On error, return QueryResponse{Error: err.Error()} (do NOT return gRPC error -- let client see the error message)

   **Status handler:**
   - Return StatusResponse with ready, model, provider, uptime_seconds (computed from startTime)

2. Create `internal/server/session.go`:
   - Helper function `assignSessionKey(clientKey, prefix string) string` that returns clientKey if non-empty, else `prefix + ":" + uuid.New().String()`
   - Used by both Chat and Query handlers

3. Create `internal/server/health.go`:
   - `StartHealthServer(addr string, readyFunc func() bool) *http.Server` -- starts HTTP server with /health and /ready endpoints
   - /health always returns 200 `{"status":"ok"}`
   - /ready returns 200 if readyFunc() is true, 503 otherwise
   - NOTE: If PicoClaw's `pkg/health.Server` is available as a dependency, prefer reusing it (see RESEARCH.md "Don't Hand-Roll"). If import issues arise, implement a minimal version matching the same contract. The key requirement is K8S-04 compatibility.

4. Run tests: `go test ./internal/server/... -timeout 30s`
