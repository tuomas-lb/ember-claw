   ```go
   package server

   import "context"

   // AgentProcessor abstracts PicoClaw's AgentLoop for testability.
   // Production: *agent.AgentLoop satisfies this.
   // Tests: inject a mock.
   type AgentProcessor interface {
       ProcessDirect(ctx context.Context, content, sessionKey string) (string, error)
   }
   ```

2. Create `internal/server/server_test.go` with tests that use a `mockProcessor` implementing `AgentProcessor`. The mock should record all calls (content, sessionKey) and return configurable responses. Tests should:
   - Start a real gRPC server on a random port using `bufconn` (in-memory listener) for speed
   - Create a gRPC client connected via bufconn
   - Test Chat bidi streaming (send message, receive response, handle EOF)
   - Test Query unary (send request, get response)
   - Test session isolation (two concurrent Chat streams, verify mock received different sessionKey values)
   - All tests should FAIL at this point because `server.go` does not exist yet (that is Plan 02)

   Use `google.golang.org/grpc/test/bufconn` for in-memory gRPC transport. Import generated code from `github.com/LastBotInc/ember-claw/gen/emberclaw/v1`.

   - These test PicoClaw's `pkg/health.Server` directly -- they should pass immediately since health.Server already works

NOTE: Tests in server_test.go will NOT compile yet (missing server.go). That is intentional -- Plan 02 creates the implementation. The tests define the contract. To make them compile but fail, create a minimal stub `internal/server/server.go` with just enough to satisfy the compiler:
