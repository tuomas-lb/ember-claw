<task type="auto">
  <name>Task 1: Dockerfile and .dockerignore for multi-stage sidecar build</name>
  <files>Dockerfile, .dockerignore</files>
  <action>
Create `Dockerfile` at project root with two stages:

**Stage 1 (builder):**
- Base: `golang:1.25-alpine`
- `RUN apk add --no-cache git`
- `WORKDIR /src`
- Copy `go.mod` and `go.sum` first, then `RUN go mod download` (layer caching)
- `COPY . .`
- `RUN CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build -ldflags="-w -s" -o /sidecar ./cmd/sidecar`

**Stage 2 (runtime):**
- Base: `alpine:3.23`
- `RUN apk add --no-cache ca-certificates tzdata`
- Create non-root user: `RUN addgroup -g 1000 picoclaw && adduser -D -u 1000 -G picoclaw -h /home/picoclaw picoclaw`
- `COPY --from=builder /sidecar /usr/local/bin/sidecar`
- `USER picoclaw`
- `EXPOSE 50051`
- `ENTRYPOINT ["sidecar"]`

Do NOT run `picoclaw onboard` in the Dockerfile (per RESEARCH anti-pattern: config comes from PVC mount at runtime).

Create `.dockerignore` excluding:
```
go.work
go.work.sum
.planning/
bin/
.git/
.ember-build-numbers
*.md
```
  </action>

  <done>Dockerfile produces a runnable linux/amd64 image. .dockerignore excludes non-build files. `docker buildx build` succeeds.</done>

<task type="auto">
  <name>Task 2: Makefile with build, push, deploy, and help targets</name>
  <files>Makefile</files>
  <action>
Create `Makefile` at project root following the umbrella repo pattern exactly. Variables:

```makefile
SERVICE_NAME := ember-claw-sidecar
DOCKERFILE := Dockerfile
IMAGE_REGISTRY := reg.r.lastbot.com
EMBER_VERSION ?=
BUILD_NUMBER_FILE := .ember-build-numbers
K8S_NAMESPACE := picoclaw
KUBECONFIG_PATH ?= /Users/tuomas/Projects/ember.kubeconfig.yaml
```

Targets (in this order):

1. **`build-eclaw`** -- `@mkdir -p bin && go build -o bin/eclaw ./cmd/eclaw && echo "Built bin/eclaw"`

2. **`build-picoclaw`** -- Build Docker image. If `EMBER_VERSION` is set, increment build counter in `.ember-build-numbers` using `sed -i.bak` (macOS-compatible) and tag as `EMBER_VERSION.BUILD_NUMBER`. Otherwise tag as `production`. Use `docker buildx build --platform linux/amd64 -f $(DOCKERFILE) -t $(IMAGE_REGISTRY)/$(SERVICE_NAME):$$IMAGE_TAG .`

3. **`push-picoclaw`** -- Push image. Read current tag from `.ember-build-numbers` (or use `production`). Run `docker push $(IMAGE_REGISTRY)/$(SERVICE_NAME):$$IMAGE_TAG`. Error if `EMBER_VERSION` is set but no build number exists (guard: `if [ ! -f $(BUILD_NUMBER_FILE) ]; then echo "Error: run make build-picoclaw first"; exit 1; fi`).

4. **`build-push-picoclaw`** -- `build-picoclaw push-picoclaw` as prerequisites.

5. **`deploy-picoclaw`** -- Prerequisites: `build-eclaw`. Interactive wizard using `read` with defaults. Support non-interactive mode via Make variable override (if `$$NAME` is already set from env, skip the prompt). Collect: NAME, PROVIDER, API_KEY (read -s for silent), MODEL. Resource limits use defaults (CPU_REQ=100m, CPU_LIM=500m, MEM_REQ=128Mi, MEM_LIM=512Mi). Shell out to:
```
./bin/eclaw deploy $$NAME \
    --provider $$PROVIDER \
    --api-key $$API_KEY \
    --model $$MODEL \
    --cpu-request $$CPU_REQ \
    --cpu-limit $$CPU_LIM \
    --memory-request $$MEM_REQ \
    --memory-limit $$MEM_LIM \
    --kubeconfig $(KUBECONFIG_PATH) \
    --namespace $(K8S_NAMESPACE)
```

6. **`help`** -- Default target. `@grep -E '^[a-zA-Z_-]+:.*?## .*$$' $(MAKEFILE_LIST) | awk ...` pattern from umbrella repo.

Each target must have a `## description` comment for help extraction. Use `@` prefix on all recipe lines to suppress echo. Use `$$` for shell variables (Make escaping).

The build-picoclaw build-number logic must handle: file doesn't exist yet, service not in file yet, service already in file (update). Use the exact `sed -i.bak ... && rm -f $(BUILD_NUMBER_FILE).bak` pattern for macOS compatibility.
